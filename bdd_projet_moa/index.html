1. Vision Stratégique (Le "Pourquoi")

1.1. Le Constat : La Fragmentation

Le suivi actuel est réparti sur plusieurs outils non connectés, engendrant un manque de vision globale, des processus incohérents et une difficulté de priorisation stratégique.

1.2. La Vision Cible : Le Système Nerveux Centralisé

Nous construisons le Système Nerveux Centralisé du pilotage projet MOA pour passer d'un pilotage réactif à un pilotage proactif, intelligent et décisionnel.

1.3. Les Piliers Fondamentaux

Clarté Radicale : Design intuitif, complexité combattue.
Friction Zéro à la Saisie : Plus simple qu'un email.
Pilotage à Double Vitesse : Gérer le Run et le Build.
La Connaissance comme Actif : Savoir capitalisé et lié.

3. Workflows & Automatisations (L'Intelligence Opérationnelle)

3.1. Templates de Saisie

Template [BUG] 🐞

Propriétés pré-remplies : Type = Bug, Priorité = Haute.

Contenu : Sections "Description", "Étapes de reproduction", "Comportement attendu/observé".

Template [STORY] ✨

Propriétés pré-remplies : Type = Story.

Contenu : Sections "Le 'Pourquoi' (User Story)" et "Le 'Quoi' (Critères d'Acceptance)".

3.2. Automatisations

  • 1. Initialisation du Ticket :

    Déclencheur : Création de page.
    Action : Statut ➔ 🔴 0. À trier.

  • 2. Clôture du Ticket :

    Déclencheur : Statut ➔ 📦 5. Terminé.
    Action : Date de résolution ➔ "Aujourd'hui".

5. Plan de Déploiement et Transition Technique

5.1. Plan de Déploiement en 4 Phases

  1. Phase 1 : Construction du Socle (BDD, propriétés, relations).
  2. Phase 2 : Création des Espaces de Travail (Hub, Portail, Roadmap).
  3. Phase 3 : Migration & Pilote (Formation et ajustements).
  4. Phase 4 : Déploiement Général (Migration complète et communication).

5.2. Spécifications pour la Transition vers une Application Web

  • Backend : Chaque BDD ➔ Table PostgreSQL. Relations ➔ Clés étrangères.
  • API : REST ou GraphQL exposant les opérations CRUD.
  • Frontend : Pages de pilotage ➔ Composants React, Angular ou Vue.js.
